UI Design Style Guidelines

The application follows a Flat + Material Designâ€“inspired system, focused on clarity, consistency, and usability for Web3 interactions. The design avoids visual noise while providing clear hierarchy and feedback for blockchain actions.

1. Design Philosophy

Flat-first design with minimal gradients

Material-inspired spacing, elevation, and motion

Function over decoration

Visual clarity for financial data

Predictable interactions and feedback

The UI prioritizes readability, trust, and speed.

2. Color System

Primary Colors

Primary Action: Green (confirm, join, withdraw)

Neutral Base: Dark gray / near-black background

Surface Cards: Slightly lighter than background

Secondary & Status Colors

Success: Green

Warning: Orange

Error: Red

Info / Inactive: Muted gray

Rules

One primary color per screen

Status colors only for state indication

No random accent usage

3. Typography

Font Style

Clean, modern sans-serif

High legibility for numbers

Hierarchy

Page Titles: Bold, large

Section Headers: Medium weight

Card Values: Bold

Labels & Hints: Smaller, muted

Rules

Max two font weights per component

Consistent line height across pages

4. Layout & Spacing

Grid System

8px base spacing unit

Consistent padding across cards and sections

Alignment

Left-aligned text for readability

Centered action buttons where appropriate

Rules

Avoid overcrowding

White space is intentional

Sections separated by spacing, not lines

5. Cards & Surfaces

Card Design

Flat surfaces with subtle elevation

Rounded corners (medium radius)

Clear internal spacing

Elevation

Light shadow only for clickable cards

No heavy drop shadows

Rules

Cards should never visually compete

One primary action per card

6. Buttons & Actions

Primary Buttons

Filled

High contrast

Used for irreversible or key actions

Secondary Buttons

Outlined or text-only

Used for navigation or secondary actions

States

Default

Hover

Disabled

Loading

Rules

Never disable without explanation

Always show transaction loading state

7. Icons & Visual Elements

Simple line or filled icons

Consistent stroke width

Icons support text, never replace it

Usage

Navigation

Status indicators

Actions (connect, withdraw)

8. Feedback & Motion

Animations

Subtle and fast

Material-inspired easing

Used only for:

Loading

Success

State transitions

Feedback

Clear confirmation on success

Clear error messaging on failure

9. Responsiveness

Mobile-first consideration

Touch-friendly components

Sidebar collapses on smaller screens

Cards stack vertically

10. Consistency Rules

Same action = same color everywhere

Same component = same layout everywhere

No page-specific styling overrides

Design tokens reused across the app

Summary

The UI design ensures:

Flat, distraction-free visuals

Material-inspired structure and motion

Consistent color and spacing system

Clear hierarchy for financial actions

Professional, production-ready look

This approach makes the app intuitive for users and credible for reviewers.
