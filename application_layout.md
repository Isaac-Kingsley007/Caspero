Application Layout Setup

Our application follows a dashboard-style layout optimized for Web3 interactions, clarity, and real-time state visibility. The layout is designed to keep wallet actions and escrow information easily accessible at all times.

1. App Shell Structure

The app is divided into three primary layout sections:

Persistent Sidebar

Top Header

Main Content Area

This structure ensures consistent navigation while allowing the core content to remain flexible and responsive.

2. Sidebar Navigation

The sidebar acts as the primary navigation hub.

Layout Characteristics

Fixed position on the left

Full viewport height

Dark theme with subtle contrast from main content

Contents

Application logo and name at the top

Navigation links:

Marketplace / Dashboard

My Escrows

History

Wallet connection status or connect button at the bottom

Behavior

Highlights the active page

Collapses into a hamburger menu on smaller screens

3. Top Header

The header provides context and wallet-level actions.

Layout Characteristics

Positioned at the top of the main content area

Minimal height with clean spacing

Sticky for easy access

Contents

Breadcrumb navigation (e.g., Dashboard / Escrow)

Primary wallet connect button

Optional network indicator or settings icon

4. Main Content Area

This is the dynamic area where all pages render.

Structure

Page title and short description at the top

Grid-based layout for cards and data sections

Scrollable without affecting sidebar or header

Design Principles

Clear spacing and alignment

Dark cards on a darker background for depth

Consistent padding and typography hierarchy

6. Responsiveness & UX

The layout is designed to be fully responsive:

Sidebar collapses on mobile

Cards stack vertically on small screens

Touch-friendly buttons and spacing

Wallet actions remain accessible at all screen sizes

7. Design Consistency

Theme

Dark mode first

Minimal gradients and soft shadows

Accent colors used only for status and actions

Typography

Clear font hierarchy

Emphasis on numeric values

Muted labels for secondary information

Summary

This layout ensures:

Fast navigation

Clear separation of concerns

Wallet-first UX

Scalable structure for future features
